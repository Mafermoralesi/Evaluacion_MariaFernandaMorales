# Respuestas de la Evaluación

## 1. Conceptos Generales

**¿Qué es un archivo Markdown y para qué sirve?**  
Un archivo Markdown es un archivo de texto que permite escribir contenido con formato simple usando símbolos. Sirve para organizar información con títulos, listas, negrita, cursiva y enlaces de una forma fácil de leer.

**¿Cuál es el propósito del archivo README.md en un repositorio?**  
*El archivo README.md sirve para explicar de qué trata el repositorio, qué contiene y cómo se usa. También ayuda a que otras personas entiendan mejor el proyecto.*

## 2. Investigación de Sintaxis Markdown

| Elemento | Sintaxis Markdown (Ejemplo) | Resultado Visual |
| :--- | :--- | :--- |
| **Título H1** | `# Texto` | # Texto |
| **Título H2** | `## Texto` | ## Texto |
| **Título H3** | `### Texto` | ### Texto |
| **Lista Desordenada** | `- Elemento` | - Elemento |
| **Lista Ordenada** | `1. Elemento` | 1. Elemento |
| **Texto en Cursiva** | `*Texto*` | *Texto* |
| **Texto en Negrita** | `**Texto**` | **Texto** |
| **Insertar un Enlace** | `[Nombre](URL)` | [Enlace](https://google.com) |

## 3. Flujo de Trabajo en GitHub Desktop

**¿Qué diferencia hay entre hacer un "Commit" y hacer un "Push"?**

* **Commit:** Es guardar los cambios que hice en mi proyecto.
* **Push:** Es subir esos cambios guardados a GitHub.

**¿Para qué sirve el botón "Fetch Origin"?**  
Sirve para revisar si hay cambios nuevos en el repositorio remoto de GitHub.
